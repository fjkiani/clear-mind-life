## What Clear Mind Life Is

Clear Mind Life is an autonomous Revenue Cycle Management (RCM) platform built specifically for mental health practices. It handles eligibility verification, clinical documentation, and claim scrubbing — the three biggest sources of administrative burden and revenue loss in mental health billing.

The platform connects to your existing EHR, your payer accounts, and your scheduling system. It doesn't replace your staff. It handles the work that shouldn't require human attention in the first place.

## The Three Agents

Clear Mind Life runs on three specialized agents. Each one owns a distinct part of the patient visit lifecycle.

### Receptionist Agent

The Receptionist Agent runs when a patient books an appointment. It sends an X12 270 eligibility transaction to the patient's payer and gets back a 271 response with coverage details: copay, deductible status, network status, and whether prior authorization is required for the planned service.

If auth is required, the agent pre-populates and submits the authorization request before the appointment date. If the patient's coverage has lapsed or the plan has a behavioral health carve-out, the agent flags it for staff review — before the patient arrives, not after the claim is denied.

### Encounter Agent

The Encounter Agent runs during and immediately after a clinical session. It transcribes the session in real time using AssemblyAI, identifies clinically relevant content, and generates a complete SOAP note draft when the session ends.

The draft includes the patient's reported symptoms (Subjective), the provider's observations (Objective), a working diagnosis with ICD-10 codes (Assessment), and the treatment plan (Plan). The provider reviews the draft and approves it — typically in under 5 minutes. The signed note triggers claim generation.

### Billing Agent

The Billing Agent runs when a claim is generated. It checks every claim against five medical coding databases before submission: ICD-10 diagnosis codes, NCCI PTP bundling edits, CPT RVU values, Medically Unlikely Edits (MUEs), and HCPCS codes.

If a claim has a coding error — missing modifier, bundled code pair, units above the MUE — the agent blocks the claim and provides a specific correction instruction. Clean claims go to the clearinghouse. Flagged claims go to the billing queue for review.

## The Medical Coding Database

The database behind the Billing Agent contains 2,505,007 records across five tables:

| Table | Records | Purpose |
|-------|---------|---------|
| ICD-10-CM | 74,719 | Diagnosis code validation and specificity checking |
| NCCI PTP | 2,387,727 | Bundling edit checks for CPT code pairs |
| CPT RVU | 15,804 | Procedure code validation and payment rate reference |
| MUE | 15,095 | Maximum units per day per CPT code |
| HCPCS | 8,727 | Supply and service codes including telehealth modifiers |

The database is updated quarterly when CMS publishes new code sets and edit tables. Every claim runs through all five tables before submission.

## How a Patient Visit Flows Through the Platform

**Step 1 — Booking:** Patient schedules an appointment. Receptionist Agent runs eligibility check. Auth request submitted if required. Practice receives eligibility summary.

**Step 2 — Session:** Provider conducts the visit. Encounter Agent transcribes in real time. SOAP note draft is generated when the session ends.

**Step 3 — Documentation:** Provider reviews and approves the SOAP note. ICD-10 codes are confirmed. CPT codes are suggested based on session type and duration.

**Step 4 — Claim generation:** Billing Agent generates the claim from the signed note. Claim runs through all five coding databases. Clean claims are submitted to the clearinghouse. Flagged claims go to the billing queue.

**Step 5 — Payment:** Clearinghouse submits the 837 claim to the payer. 835 remittance is received and matched to the claim. Payment is posted to the patient's account.

The entire flow — from booking to claim submission — happens without manual data entry at any step.
